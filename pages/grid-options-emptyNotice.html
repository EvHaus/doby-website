options.emptyNotice

Displays a "No Results Found" message when no data is available to be rendered (ie. the data set is empty). This is recommended if you're using a remote data source or if you are making use of the filtering options. Otherwise the grid will suddenly become empty with no user feedback.

You can use "true" to display a default text message, or a function that returns an HTML string to display a custom overlay. The empty notice is rendered in an overlay, just like grid.setOverlay().

Type and Default

(boolean), (string): Default true.

Examples
// Disable empty notices
{
    emptyNotice: false
}

// Custom empty notice
{
    emptyNotice: function () {
        return "Custom message";
    }
}
