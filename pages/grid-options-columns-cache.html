options.columns.cache

When used in combination with the options.columns.postprocess option, enabling this option will build a cache of the rendered cells in this column and use that cache when re-rendering the grid view. This prevents constant processing of cells during scrolling and resizing.

Depending on the complexity of your grid and cell data, this could quickly use up a lot of memory. Use at your own risk.

Type and Default

(boolean): Default false.

Example
{cache: true}
